The Authentication Handshake

For an agent to securely access a tool, it must complete a precise, token-based authentication flow. Every step is critical to ensure only authorized identities can communicate.

1

Agent Invocation

A user prompt triggers the agent, which determines it needs to call the external tool.

2

Token Generation

The agent's code requests a Google-signed OIDC ID Token, specifying the tool's URL as the unique `audience`.

3

Authenticated Request

The agent sends its request to the tool, including the token in the `Authorization: Bearer ` header.

4

Token Validation

Cloud Run's frontend intercepts the request, validates the token's signature, and confirms its `audience` matches the service URL.

5

IAM Authorization

Cloud Run checks if the agent's identity (from the token) has the required `run.invoker` permission in its IAM policy.

6

Access Granted/Denied

If all checks pass, the request proceeds. If not, Cloud Run returns a `403 Forbidden` error, and the tool is never reached.

Identity & The Principle of Least Privilege

Security isn't just about who gets in; it's about granting them the absolute minimum permissions required. Overly broad roles create unnecessary risk.

Building the Network Fortress

Identity is the first line of defense. A private network is the second. By isolating services from the public internet, we create a hardened perimeter.

A Defense-in-Depth Strategy

This multi-layered approach ensures that even if one layer is misconfigured, others provide backup protection. Traffic is forced onto a secure, private path from start to finish.

Implementation Highlights

Authenticated Call

In the agent's Python code, explicitly fetching a token with the correct `audience` is the most critical step for security.

id_token.fetch_id_token(
  auth_req, 
  MCP_TOOLBOX_URL
)

Secure Deployment

When deploying the Cloud Run tool, locking down ingress is a simple but powerful command-line flag.

gcloud run deploy ... \
  --ingress=internal

Assigning Identity

When creating the agent, you must attach the dedicated service account you created to act as its identity.

ReasoningEngine.create(
  ...,
  service_account=...
)
